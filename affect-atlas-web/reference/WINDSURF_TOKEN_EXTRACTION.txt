WHAT THIS IS
------------
Each emotion has a Stitch-generated HTML screen with a Tailwind CSS config
containing color tokens, fonts, border radii, and motion values. These need
to be written into the TypeScript token file that the app uses at runtime.


SOURCE FILES (read from these)
------------------------------
Each file is at: stitch-exports/v2-{NN}-{emotion}/code.html
Look for the <script id="tailwind-config"> block. It contains a JS object
with colors, fontFamily, borderRadius, and spacing.

Motion values (easing, duration) are in a <style> block, typically in a
class named after the emotion's motion character (e.g. .bouncy-ease,
.deflating-ease). Extract the cubic-bezier and duration from there.

FONT KEY QUIRKS IN HTML
------------------------
Most files use: fontFamily.headline and fontFamily.body
Two exceptions:
  v2-19-depression uses "display" instead of "headline"
  v2-24-jealousy uses "serif" and "sans" instead of "headline" and "body"
Map these to headlineFont and bodyFont regardless of key name.


ENTRY FORMAT
------------
Each emotion needs exactly 3 entries (mild, moderate, intense).
Each Stitch HTML file is ONE screen -- it represents the MODERATE intensity.

For mild: desaturate/lighten the colors slightly from the HTML values.
For intense: saturate/darken the colors slightly from the HTML values.
Keep the same fonts and corner style across all 3 intensities.

Here is the exact TypeScript shape for each entry:

  {
    emotion: 'emotionname',
    intensity: 'mild',        // or 'moderate' or 'intense'
    source: 'v2-NN-emotionname',
    placeholder: false,
    isDark: boolean,           // true if background is dark, false if light
    colors: {
      primary: '#hex',         // from Tailwind "primary"
      secondary: '#hex',       // from Tailwind "secondary"
      accent: '#hex',          // from Tailwind "tertiary" (mapped to accent)
      background: '#hex',      // from Tailwind "surface" or body bg
      surface: '#hex',         // from Tailwind "surface-container"
      text: '#hex',            // from Tailwind "on-surface"
      textSecondary: '#hex',   // from Tailwind "on-surface-variant"
      palette: ['#hex','#hex','#hex','#hex','#hex'],
        // 5 representative colors: [accent, primary, secondary, surface, background]
    },
    typography: {
      headlineFont: 'Font Name',   // from fontFamily.headline[0]
      bodyFont: 'Font Name',       // from fontFamily.body[0]
    },
    shapes: {
      borderRadius: N,             // from borderRadius.DEFAULT or borderRadius.md (parse px to number)
      borderRadiusLg: N,           // from borderRadius.lg (parse px to number)
      borderRadiusXl: N,           // use borderRadius.full? or 2x lg. If not available, use 2x lg
      cornerStyle: 'rounded',      // 'rounded' if radius >= 12, 'smooth' if 4-11, 'sharp' if 0-3, 'mixed' if varies
    },
  },


MAPPING CHEAT SHEET: Tailwind config key -> TypeScript field
------------------------------------------------------------
  colors.primary            -> colors.primary
  colors.secondary          -> colors.secondary
  colors.tertiary           -> colors.accent
  colors.surface            -> colors.background  (yes, surface maps to background)
  colors.surface-container  -> colors.surface
  colors.on-surface         -> colors.text
  colors.on-surface-variant -> colors.textSecondary
  fontFamily.headline[0]    -> typography.headlineFont
  fontFamily.body[0]        -> typography.bodyFont
  borderRadius.DEFAULT      -> shapes.borderRadius (strip "px", convert to number)
  borderRadius.lg           -> shapes.borderRadiusLg (strip "px", convert to number)


isDark DETERMINATION
--------------------
Look at the background color. If it is dark (luminance < 0.3), isDark = true.
Quick rule: if the hex starts with #0, #1, #2, or #3 it is dark.
Reference from existing entries and emotion table:

  DARK: fear, anger, sadness, surprise, disgust, confusion, appalled,
        guilt, awe, depression, loneliness, jealousy, fright, frustration,
        pride, remorse, aggressiveness, alarm
  LIGHT: joy, trust, anticipation, contentment, inspiration,
         embarrassment, love, submission, disappointment, contempt, optimism


MILD / MODERATE / INTENSE STRATEGY
-----------------------------------
The HTML gives you the MODERATE values directly. For mild and intense:

MILD (lighter/softer version):
  - background: lighten by ~5-10% (e.g. if moderate bg is #f0f0f2, mild is #f4f4f6)
  - primary: desaturate slightly or use primary-container color from HTML
  - Keep fonts, corner style, and shapes identical to moderate
  - Keep same isDark value

INTENSE (stronger/saturated version):
  - background: darken slightly or use a more saturated variant
  - primary: use a more saturated or contrasted version
  - accent: pick the most vivid color from the HTML palette
  - Keep fonts, corner style, and shapes identical to moderate
  - Keep same isDark value

If you cannot confidently derive mild/intense, just use the SAME values
as moderate for all 3. Getting the moderate values right is the priority.


EMOTION TYPE
------------
The EmotionType union in types/emotion.ts currently only includes:
  'joy' | 'trust' | 'fear' | 'surprise' | 'sadness' | 'disgust' |
  'anger' | 'anticipation' | 'love' | 'submission' | 'alarm' |
  'disappointment' | 'remorse' | 'contempt' | 'aggressiveness' | 'optimism'

The 13 new emotions (awe, confusion, depression, etc.) are NOT in this union.
DO NOT try to add them to EmotionType. Instead, use a type assertion:
  emotion: 'awe' as any,

This is a known workaround documented in MEMORY.md. The union expansion
is a separate task.
